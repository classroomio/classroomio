ClassroomIO is open source, and you can run the whole platform on your own infrastructure. Every
option below runs the same stack — the API, the dashboard, and the background jobs worker, backed by
Postgres and Redis.

Pick the path that matches where you want it to run.

## Which should I choose?

| Option                                | Best for                                                              |
| ------------------------------------- | --------------------------------------------------------------------- |
| [Docker](/self-hosted/docker)         | Full control on your own server, or as the base for another platform. |
| [Railway](/self-hosted/railway)       | Getting a working instance up with the least infrastructure work.     |
| [Coolify](/self-hosted/coolify)       | Running a PaaS on your own VPS, managing several apps alongside it.   |

:::note[You need an SMTP provider]
Signup verification, password resets, and invite emails all send over SMTP. Whichever option you
pick, have mail credentials ready before you start.
:::

## What you'll need

- A **Postgres** database and a **Redis** instance (both are included in the Docker and Coolify compose files).
- An **SMTP provider** for transactional email.
- Object storage (**MinIO** or S3-compatible) if you want file and video uploads.
- *Optional:* an **OpenAI API key** for transcription and AI course generation.

For the full environment variable reference, see the
[Docker guide](/self-hosted/docker), which documents every variable the stack reads.
