However you deploy ClassroomIO — [Docker](/self-hosted/docker), [Railway](/self-hosted/railway), or
[Coolify](/self-hosted/coolify) — the stack reads the **same set of environment variables**. This
section is the single source of truth for what each group of variables does. The provider guides show
you *where* to set them for that platform; these pages explain *what* to set and *why*.

## How to read these pages

ClassroomIO is configured entirely through environment variables. On Docker they live in one root
`.env` file that every `docker compose` command reads via `--env-file .env`; on Railway and Coolify you
set the same variables on each service.

:::note[Value = required, blank = optional]
Throughout these pages and the provider guides, a variable shown **with a value** is one you should
set for a working deployment. A variable left **blank after `=`** is optional — fill it in only if you
use that feature. Leaving an optional variable blank never causes a build or boot error.
:::
