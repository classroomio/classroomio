title: Environment & secrets
description: Generate the two secrets ClassroomIO needs to boot — BETTER_AUTH_SECRET and PRIVATE_SERVER_KEY.
---

ClassroomIO needs two secrets before it will run. Generate them once, keep them safe, and reuse the
same values across the services that need them.

## The two secrets

```zsh title="Terminal"
openssl rand -hex 32   # BETTER_AUTH_SECRET
openssl rand -hex 32   # PRIVATE_SERVER_KEY   (must be identical in the api and dashboard)
```

| Variable | What it's for | Where it goes |
|----------|---------------|---------------|
| `BETTER_AUTH_SECRET` | Signs authentication sessions. | The **api** service. |
| `PRIVATE_SERVER_KEY` | Authenticates server-side dashboard → API requests. | **Both** the api and dashboard — the value must be **identical** in each. |

:::danger[The stack won't start without them]
The compose file **fails fast** if `BETTER_AUTH_SECRET` or `PRIVATE_SERVER_KEY` is empty — the
containers refuse to boot rather than run with a predictable, guessable session-signing secret. Set
both to a strong random value (e.g. `openssl rand -hex 32`).
:::

:::note[Who generates them]
Only the source-build path's `run-docker-full-stack.sh` auto-generates these when missing. The
pre-built image path, Railway, and Coolify do **not** run that script — you generate the secrets
yourself and paste them in.
:::

## Database credentials

Postgres runs inside the stack (or as a managed plugin on Railway). Set a strong password — the
defaults are for local use only.

| Variable | Description | Default |
|----------|-------------|---------|
| `POSTGRES_DB` | Database name | `classroomio` |
| `POSTGRES_USER` | Database user | `postgres` |
| `POSTGRES_PASSWORD` | Database password — set a **strong** value in production | `postgres` |

## Mode & routing basics

| Variable | Description | Default |
|----------|-------------|---------|
| `PUBLIC_IS_SELFHOSTED` | Self-hosted mode flag | `true` |
| `PRIVATE_SERVER_URL` | Internal API URL for server-side rendering (over the container network) | `http://api:3081` |
| `DASHBOARD_ORIGIN` | Public dashboard origin, used as the auth base URL and in invite/email links | `http://localhost:3082` |

:::warning[Use an `https://` origin in production]
`DASHBOARD_ORIGIN` (and `ORIGIN` on the dashboard) must be the real `https://` URL your users visit —
auth session cookies are `Secure`, so login silently fails over plain HTTP on any public domain. See
[Self Hosting → HTTPS is required in production](/self-hosted#https-is-required-in-production).
:::
