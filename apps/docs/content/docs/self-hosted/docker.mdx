Docker is one way to self-host ClassroomIO. There are two paths:

- **Run from pre-built images (recommended)** — pull the published `classroomio/{api,dashboard,jobs}` images. You only need one compose file and a `.env`; no repository clone and no heavy local build.
- **Build from source** — clone the repo and build the images yourself (needs ~8 GB RAM for the dashboard build).

![Selfhost ClassroomIO](/selfhost-classroomio.webp)

## Run from Pre-built Images (recommended)

### Prerequisites

- Docker installed and running.

### Steps

1. Download the pull-based compose file and an env template:

```zsh title="Terminal"
curl -O https://raw.githubusercontent.com/classroomio/classroomio/main/docker-compose.images.yaml
curl -o .env https://raw.githubusercontent.com/classroomio/classroomio/main/.env.example
```

2. Edit `.env`. At minimum set `CIO_VERSION` (pin a release), `DASHBOARD_ORIGIN`, the auth secrets, and SMTP:

```zsh title=".env"
CIO_VERSION=1.4.2                 # pin a release (not "latest") so upgrades are deliberate
DASHBOARD_ORIGIN=https://app.yourdomain.com
BETTER_AUTH_SECRET=<openssl rand -hex 32>
PRIVATE_SERVER_KEY=<openssl rand -hex 32>   # must be identical in the api and dashboard
POSTGRES_PASSWORD=<strong password>
# Email (required for signup/reset/invites)
SMTP_HOST=
SMTP_PORT=
SMTP_USER=
SMTP_PASSWORD=
SMTP_SENDER=
```

<Callout type="error" title="Set strong secrets — the stack won't start without them">
  This path does **not** run `run-docker-full-stack.sh`, so secrets are not auto-generated. You must
  set `BETTER_AUTH_SECRET` and `PRIVATE_SERVER_KEY` yourself — e.g. `openssl rand -hex 32`. The
  compose file fails fast with a clear error if either is empty, so it never boots with a
  predictable session-signing secret.
</Callout>

3. Pull and start the stack (MinIO is included via the `minio` profile):

```zsh title="Terminal"
docker compose -f docker-compose.images.yaml --env-file .env --profile minio pull
docker compose -f docker-compose.images.yaml --env-file .env --profile minio up -d
```

4. Verify:

```zsh title="Terminal"
curl -sS http://localhost:3081/    # API — returns JSON
curl -I  http://localhost:3082/    # Dashboard — returns 200
```

- Dashboard: http://localhost:3082
- API: http://localhost:3081
- MinIO Console: http://localhost:9001 (log in with `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD` from your `.env`)

## Build from Source

Use this if you want to build the images locally (e.g. to run code changes).

### Prerequisites

- Docker Desktop installed and running
- `git` installed
- ~8 GB RAM available for the dashboard build

### Steps

1. Clone the repository:

```zsh title="Terminal"
git clone https://github.com/classroomio/classroomio.git
cd classroomio
```

2. Copy the env template and edit it:

```zsh title="Terminal"
cp .env.example .env
```

Open `.env` and set `DASHBOARD_ORIGIN` for your domain. For local development the defaults work as-is — the startup script generates `BETTER_AUTH_SECRET` and `PRIVATE_SERVER_KEY` when they are missing, and randomizes the MinIO credentials.

3. Build and start the full stack:

```zsh title="Terminal"
./run-docker-full-stack.sh --build
```

The script builds and starts Postgres, Redis, MinIO (object storage), the API, the dashboard, and the background **jobs** worker.

<Callout title="Default is pull, not build">
  Run without `--build` and the script **pulls** the published `classroomio/*` images at the
  `CIO_VERSION` in your `.env` instead of building locally — that's the faster default path.
</Callout>

## Upgrading to a New Version

ClassroomIO publishes versioned images. `CIO_VERSION` controls which tag you run:

- Pin an exact release (e.g. `1.4.2`) in production so a merge to `main` never restarts you onto changed images unattended.
- `latest` = the freshest `main` build — it moves on every merge (rolling; may include unreleased code). Version tags (`1.4.2`, `1.4`, `1`) are the immutable, pinned releases.

**Pre-built images path** — bump `CIO_VERSION` in `.env`, then re-pull and recreate:

```zsh title="Terminal"
docker compose -f docker-compose.images.yaml --env-file .env --profile minio pull
docker compose -f docker-compose.images.yaml --env-file .env --profile minio up -d
```

**Rollback** — set `CIO_VERSION` back to the previous release and run the same two commands.

**Build-from-source path** — `git pull`, then re-run `./run-docker-full-stack.sh --build` to rebuild.

<Callout type="warn" title="Back up before upgrading">
  Database schema migrations run automatically when the API starts. Back up your Postgres volume
  (and object storage) before a major upgrade, and avoid `docker compose down -v` / `docker volume
  prune`, which delete your data volumes.
</Callout>

## One `.env`, One Command

Docker self-hosting uses a single root `.env` file. Every `docker compose` command reads it via `--env-file .env` — you never need separate env files per service. See [`.env.example`](https://github.com/classroomio/classroomio/blob/main/.env.example) for the full template.

## Environment Variables

### Required

| Variable | Description | Default |
|----------|-------------|---------|
| `POSTGRES_DB` | Database name | `classroomio` |
| `POSTGRES_USER` | Database user | `postgres` |
| `POSTGRES_PASSWORD` | Database password | `postgres` |
| `BETTER_AUTH_SECRET` | Auth session signing secret (required — stack won't start if empty) | — |
| `PRIVATE_SERVER_KEY` | Dashboard-to-API auth key (must be identical in api + dashboard) | Auto-generated by the script |
| `PUBLIC_IS_SELFHOSTED` | Self-hosted mode flag | `true` |
| `PRIVATE_SERVER_URL` | Internal API URL for SSR (Docker network) | `http://api:3081` |
| `DASHBOARD_ORIGIN` | Dashboard origin for invite links, email links, etc. | `http://localhost:3082` |

### CSP Domains (Runtime)

CSP domain variables are **runtime** — set them in your `.env`. No image rebuild is needed when changing them.

| Variable | Description |
|----------|-------------|
| `ALLOWED_EXTERNAL_DOMAINS` | Overrides all per-directive CSP vars (comma-separated) |
| `CSP_SCRIPT_SRC_DOMAINS` | Domains allowed in `script-src` |
| `CSP_STYLE_SRC_DOMAINS` | Domains allowed in `style-src` |
| `CSP_CONNECT_SRC_DOMAINS` | Domains allowed in `connect-src` |
| `CSP_FRAME_SRC_DOMAINS` | Domains allowed in `frame-src` |
| `CSP_FONT_SRC_DOMAINS` | Domains allowed in `font-src` |
| `CSP_MEDIA_SRC_DOMAINS` | Domains allowed in `img-src` and `media-src` |

<Callout type="warn" title="CSP Domains">
  When `PUBLIC_IS_SELFHOSTED=true`, the dashboard starts with only `'self'` allowed. Set `ALLOWED_EXTERNAL_DOMAINS` (or the per-directive vars) with your CDN, video, and font domains. The API does not need to be added for normal dashboard traffic because requests are proxied same-origin. Since these are runtime env vars, you can change them by restarting the container — no rebuild required.
</Callout>

### Optional — Email

Configure SMTP. Without email config, the app works but cannot send invites, password resets, or notifications.

| Variable | Description |
|----------|-------------|
| `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_SENDER` | SMTP server |

### Optional — Object Storage

The bundled **MinIO** is included by default and the startup script auto-configures it — you only need to change these for custom credentials or an external S3-compatible service.

| Variable | Default |
|----------|---------|
| `MINIO_ROOT_USER` | randomized by the script (`minioadmin` if you skip it) |
| `MINIO_ROOT_PASSWORD` | randomized by the script (`minioadmin` if you skip it) |
| `OBJECT_STORAGE_ENDPOINT` | `http://minio:9000` |
| `OBJECT_STORAGE_PUBLIC_ENDPOINT` | `http://localhost:9000` |
| `OBJECT_STORAGE_ACCESS_KEY_ID` | matches `MINIO_ROOT_USER` |
| `OBJECT_STORAGE_SECRET_ACCESS_KEY` | matches `MINIO_ROOT_PASSWORD` |
| `OBJECT_STORAGE_FORCE_PATH_STYLE` | `true` |
| `OBJECT_STORAGE_MEDIA_PUBLIC_BASE_URL` | `http://localhost:9000/media` |

<Callout title="Using AWS S3 or Cloudflare R2 instead of MinIO">
  The `OBJECT_STORAGE_*` variables speak the S3 API, so they work with any S3-compatible provider.
  To use Cloudflare R2 or AWS S3, point `OBJECT_STORAGE_ENDPOINT` at the provider's endpoint and set
  the matching access key / secret, then start without the bundled MinIO via
  `./run-docker-full-stack.sh --no-minio`. See the [Cloudflare Setup](/docs/contributor-guides/cloudflare-setup) guide for R2.
</Callout>

### Optional — Integrations

| Variable | Description |
|----------|-------------|
| `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | Google OAuth |
| `PRIVATE_APP_HOST`, `PRIVATE_APP_SUBDOMAINS` | Multi-subdomain routing |
| `UNSPLASH_API_KEY` | Unsplash image search |
| `OPENAI_API_KEY` | Transcription + AI course generation (used by the jobs worker) |
| `LICENSE_KEY` | Enterprise features (SSO, token-auth, no-tracking) |
| `PUBLIC_APP_TITLE`, `PUBLIC_APP_DESCRIPTION`, `PUBLIC_OG_IMAGE_URL` | Custom branding |

### Optional — Upload file size limits

Set these in the root `.env` to raise or lower upload caps (values are in **megabytes**). Compose passes them through to the `api` and `dashboard` containers. Unset = built-in defaults. On [Railway](/docs/self-hosted/railway) or [Coolify](/docs/self-hosted/coolify), set the same variables on both the API and dashboard services.

| Variable | Default | Applies to |
|----------|---------|------------|
| `UPLOAD_MAX_DOCUMENT_MB` | 5 | Course lesson documents |
| `UPLOAD_MAX_IMAGE_MB` | 5 | Avatars, media library, editor images |
| `UPLOAD_MAX_VIDEO_MB` | 800 | Lesson video uploads |
| `UPLOAD_MAX_EXERCISE_FILE_MB` | 2 | Student assignment file-upload questions |
| `UPLOAD_MAX_AGENT_DOCUMENT_MB` | 5 | AI assistant document uploads |
| `UPLOAD_MAX_LANDING_IMAGE_MB` | 0.5 | Org landing page cover images |
| `UPLOAD_MAX_THUMBNAIL_MB` | 5 | Media thumbnail uploads |

<Callout type="warn" title="Reverse proxy limits">
  If you raise these values, also increase your reverse proxy body-size limit (e.g. nginx
  `client_max_body_size`) so large uploads are not rejected before they reach ClassroomIO.
</Callout>

## Common Commands

```zsh title="Terminal"
# Start the full stack (pulls pre-built images by default)
./run-docker-full-stack.sh

# Build from source instead of pulling
./run-docker-full-stack.sh --build

# Start without MinIO (requires an external S3-compatible store)
./run-docker-full-stack.sh --no-minio
```

See [docker/docs/SELF_HOST.md](https://github.com/classroomio/classroomio/blob/main/docker/docs/SELF_HOST.md) for the full Docker reference including troubleshooting.

## Troubleshooting: "No space left on device"

```zsh title="Terminal"
docker system df              # check usage
docker builder prune -f       # free build cache
docker image prune -a -f      # free unused images
docker container prune -f     # free stopped containers
```

<Callout type="warn" title="Volume prune deletes data">
  Avoid `docker volume prune` after `docker compose ... down` — it can remove your database and storage volumes.
</Callout>

Then restart: `./run-docker-full-stack.sh`
