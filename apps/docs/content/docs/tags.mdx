Tags in ClassroomIO help teams structure course catalogs and improve discovery across admin
and learner experiences.

## Scope and Ownership

- Tags are organization-scoped.
- A tag cannot be shared across organizations.
- Each tag belongs to a tag group (facet/category).

## Roles and Permissions

- Admin:
  - Create and update tag groups.
  - Create and update tags.
  - Configure tag properties used for filtering.
- Tutor:
  - Assign existing tags to managed courses.
- Student:
  - Filter and discover courses using tags.

## Admin Workflow

1. Open `Org -> Tags` at `/org/[slug]/tags`.
2. Create a tag group (for example `Difficulty`, `Technology`, `Job Offering`).
3. Create tags inside a group.
4. Choose a tag color from the fixed 10-color popover palette.
5. Use the `Total Courses` value in the table to deep-link into filtered org courses:
   `/org/[slug]/courses?tags=<tagSlug>`.

## Filtering Surfaces

- Org Courses:
  - Supports URL-based filtering with `?tags=<comma-separated-tag-slugs>`.
- Public `/courses` (org site/custom domain):
  - Displays published courses.
  - Supports tag filtering with URL query params.

## URL Query Format

- Single tag:
  - `/org/acme/courses?tags=beginner`
- Multiple tags:
  - `/org/acme/courses?tags=beginner,react`
- Public catalog:
  - `/courses?tags=beginner,react`

## API Notes

Current tags-related endpoints include:

- `GET /organization/tags`
- `POST /organization/tags`
- `PUT /organization/tags/:tagId`
- `GET /organization/tags/public`
- `GET /course/:courseId/tags`
- `PUT /course/:courseId/tags`
- `GET /organization/courses/public?siteName=<siteName>&tags=<...>`
