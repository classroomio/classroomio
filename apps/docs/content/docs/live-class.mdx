A **Live class** course runs around scheduled sessions you host on your own video tool (Zoom, Google Meet, Teams).
ClassroomIO handles everything around the session: the join link, the schedule, automatic reminders, a calendar invite,
and a one-click way for students to join.

## Set the course type

Set the course type to **Live class** in the course's **Settings** tab. See [Course Types](/docs/course-types)
for what each type does.

## Schedule the session

Each session lives on a lesson. Create the lesson for what you'll run live, then open its **Settings** tab and fill in
the **Live session** section.

![Lesson Settings tab showing the Live session fields for join link, date and time, and timezone](/live-class-session-settings.webp)

<Steps>

<Step>
### Create a lesson to run live

Add a lesson to the course for the session you'll hold live (for example, the webinar topic). See
[Creating Your First Course](/docs/create-first-course#adding-lessons) for adding lessons.
</Step>

<Step>
### Add the join link

In **Join link (Zoom, Meet, etc.)**, paste the meeting URL students will use to join.
</Step>

<Step>
### Set the date and time

Pick the **Date and time** of the session.
</Step>

<Step>
### Choose the timezone

Set the **Timezone** for the session. Reminders and the calendar invite display the time in this timezone, so students
always see when the session starts in the zone you intend, regardless of where you or they are located.
</Step>

<Step>
### Save the session

Click **Save session**.
</Step>

</Steps>

## What students get

Once a session is scheduled, enrolled students automatically receive:

- **A calendar invite when they enroll.** The welcome email for a Live class course attaches a calendar invite (`.ics`)
  for the next session, so students can add it to their calendar right away.
- **Reminder emails.** One about 24 hours before and one about 1 hour before the session, each with the join link. The
  24-hour reminder also includes the calendar invite. Once added, the calendar reminds the student on its own.
- **A Join session button.** On their **LMS Home**, students see a **Join session** button on the course while a session
  is upcoming, which opens the join link directly.

![LMS Home Live Sessions card with a Join session button](/live-class-lms-card.webp)

In the course content sidebar, a live-session lesson shows a red live (broadcast) icon, so you and your students can spot
it at a glance.

![Student live session card showing the countdown and Add to calendar button](/live-class-student-card.webp)

<Callout type="info">
Reminders are sent to enrolled students only. The times shown in emails and the calendar invite use the **Timezone** you
set on the session. The welcome email, reminders, and updates all reference the same calendar event, so students never
get a duplicate on their calendar.
</Callout>

## Change the session time

If you change the **Date and time**, **Join link**, or **Timezone** of a session that students are already enrolled in,
saving asks you to confirm with **Notify students of the change?**. Choosing **Save and notify** emails every enrolled
student about the update with a fresh calendar invite that updates the existing event on their calendar instead of
creating a duplicate.

![Confirmation dialog asking to notify students of the session change](/live-class-change-time.webp)

<Callout type="warn">
**Save and notify** emails all enrolled students. Use it when the time or link actually changed. For a first-time
schedule there is nothing to notify, so it saves without sending.
</Callout>

## After the session

Pair the session with a post-session quiz and a certificate to round out the workflow:

- Add a graded quiz with [How to Create an Exercise](/docs/create-exercise), then use **Notify students**
  to send everyone a link to take it.
- Issue completion certificates with [How to Create Certificates](/docs/certificates).
